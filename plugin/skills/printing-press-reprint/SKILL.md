# printing-press-reprint

Reprint an existing CLI that was previously generated by the Hermes Printing Press.
Preserves custom business logic while regenerating the scaffold and quality gates
under the current binary version.

## Sibling skills

- `skill_view("printing-press-hermes:printing-press")` — generate a brand-new CLI.

## When to use

User says any of:
- "reprint <slug>"
- `/printing-press-reprint <slug>`
- "upgrade the <slug> CLI to the latest press"

## Pre-flight

Same binary check as the main skill:

```bash
HERMES_PRESS="${HERMES_PRESS_BINARY:-hermes-press}"
if ! command -v "$HERMES_PRESS" >/dev/null 2>&1; then
  echo "[setup-error] hermes-press not found — install it first."
  exit 1
fi
"$HERMES_PRESS" version
```

## Step 1 — Locate existing CLI

```bash
API_SLUG="$1"
LIBRARY_DIR="$HOME/hermes-press/library/$API_SLUG"
ls "$LIBRARY_DIR" 2>/dev/null || echo "[not-found] $LIBRARY_DIR does not exist"
```

If the library entry is missing, ask the user: print fresh with
`skill_view("printing-press-hermes:printing-press")` instead?

## Step 2 — Snapshot customizations

Before regenerating, diff the existing library copy against a clean scaffold to
identify hand-written business logic:

```bash
# Show files that differ from a pure scaffold
find "$LIBRARY_DIR" -name "*.go" | while read f; do
  wc -l "$f"
done
```

List the files that contain custom logic the user wants to preserve. Ask the user
to confirm before continuing.

## Step 3 — Re-research

Run Phase 1 research with `--force` to refresh `research.json` (new API features,
changed auth, updated competitors):

```bash
"$HERMES_PRESS" research "$API_SLUG" --json
```

## Step 4 — Regenerate scaffold

```bash
"$HERMES_PRESS" generate "$API_SLUG" \
  --run-dir "$HOME/hermes-press/runs/$API_SLUG" --json
```

## Step 5 — Reapply customizations

Re-apply the customizations identified in Step 2 to the new scaffold. This is a
manual merge step — do it file by file, reading the old and new versions carefully.

## Step 6 — Verify and publish

Follow Phases 4–5 from `skill_view("printing-press-hermes:printing-press")`:
verify with `--strict`, then publish.

## Pitfalls

- **Overwriting custom logic**: always snapshot before regenerating. If Step 2 is
  skipped, custom work is irrecoverable.
- **Stale research.json**: the reprint always re-runs Phase 1. If you want to reuse
  old research, pass the existing `research.json` path and skip to Phase 2.
